# Contributing

## Development setup
1. Clone the repository.
2. Run `npm install`.
3. Run `npm start` for interactive mode.
4. Run `npm test` before opening a pull request.

## Pull request rules
- Keep changes focused and small.
- Include rationale in commit messages.
- Update `README.md` for behavior or CLI changes.
- Avoid unrelated formatting-only edits.

## Issue reports
When reporting bugs, include:
- Node version
- OS details
- Exact command used
- Expected behavior vs actual behavior
